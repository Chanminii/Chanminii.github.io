## üöÄ Quick Start

### Option 1: GitHub Pages (Recommended)

1. **Create a new repository** on GitHub named `Chanminii.github.io` (replace with your username)

2. **Upload files**:
   - Upload `index.html`, `styles.css`, and `script.js` to the repository
   - Or clone and push:
     ```bash
     git clone https://github.com/Chanminii/Chanminii.github.io.git
     cd Chanminii.github.io
     # Copy your files here
     git add .
     git commit -m "Initial portfolio commit"
     git push origin main
     ```

3. **Enable GitHub Pages**:
   - Go to repository Settings ‚Üí Pages
   - Select source: Deploy from main branch
   - Your site will be live at `https://chanminii.github.io`

### Option 2: Local Development

1. **Download all files** to a folder
2. **Open `index.html`** in your browser
3. That's it! No build process required

### Option 3: Deploy to Netlify/Vercel

1. **Drag and drop** the folder to Netlify Drop or Vercel
2. Your site will be live instantly with a custom URL

## üé® Customization

### Update Your Information

Open `index.html` and update:
- Name and title in the hero section
- About section text
- Work experience details
- Projects (add/remove as needed)
- Skills and certifications
- Contact information

### Change Colors

Open `styles.css` and modify CSS variables:
```css
:root {
    --primary-color: #667eea;  /* Main theme color */
    --primary-dark: #764ba2;   /* Dark variant */
    /* ... other colors */
}
```

### Add Your Photo

1. Add your image file to the repository
2. In `index.html`, update the hero section:
```html
<div class="hero-image">
    <img src="your-photo.jpg" alt="Chanmini Weerakoon">
</div>
```

### Modify Projects

In `index.html`, find the projects section and update:
- Project titles
- Descriptions
- Technologies used
- GitHub links

## üêõ Troubleshooting

**Portfolio not showing?**
- Check repository name is `username.github.io`
- Ensure `index.html` is in the root directory
- Wait a few minutes for GitHub Pages to build

**Styles not loading?**
- Check file names match exactly (case-sensitive)
- Ensure all files are in the same directory
- Clear browser cache

**Images not showing?**
- Use relative paths: `./image.jpg`
- Ensure image files are uploaded to repository
